## 四、部署步骤 (简版)

### 方式一: 直接部署 (快速)

```bash
# 1. 上传代码到服务器
scp -r mirothinker-service/ user@your-server:/opt/

# 2. SSH 到服务器
ssh user@your-server

# 3. 安装依赖
cd /opt/mirothinker-service/backend
pip3 install -r requirements.txt

# 4. 配置环境变量
cp /opt/mirothinker-service/.env.example /opt/mirothinker-service/.env
nano /opt/mirothinker-service/.env  # 填入 API 密钥

# 5. 启动服务
nohup python3 -m uvicorn main:app \
    --host 0.0.0.0 \
    --port 8000 \
    --workers 2 \
    > /var/log/mirothinker.log 2>&1 &

# 6. 配置 Nginx (参考 nginx.conf)
sudo cp nginx.conf /etc/nginx/conf.d/mirothinker.conf
sudo nginx -t && sudo systemctl reload nginx
```

### 方式二: Docker 部署 (推荐)

```bash
# 1. 构建镜像
docker build -t mirothinker-service:latest .

# 2. 启动容器
docker run -d \
    --name mirothinker \
    -p 8000:8000 \
    --env-file .env \
    mirothinker-service:latest

# 3. 验证
curl http://localhost:8000/api/health
```

## 七、风险评估

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|----------|
| API 密钥泄露 | 高 | 低 | 使用环境变量，不提交 .env |
| 服务器过载 | 高 | 中 | 添加 Rate Limiting，监控 CPU/内存 |

| 搜索结果不足 | 中 | 低 | 固定渠道 + 通用搜索兜底 |
| 并发冲突 | 低 | 低 | 使用 asyncio 异步处理 |
