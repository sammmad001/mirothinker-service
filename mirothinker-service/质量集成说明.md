# MiroThinker 质量提升集成说明

> 版本: v2.0 | 2026-05-11

## 概述

本次更新将质量提升方案中的核心功能集成到了 `backend/main.py` 中，实现了以下模块：

## 已集成模块

### 1. 固定信息渠道 (FixedChannelSearch)

**位置**: `main.py` 第 100-130 行

**功能**:
- 定义 4 类核心信源：学术、技术、新闻、数据
- 按优先级搜索：L1 核心信源 → L3 通用搜索兜底
- 使用 `site:` 操作符精准定位高质量信源

**配置**:
```python
CORE_SOURCES = {
    "academic": {...},  # arXiv, PubMed, Scholar
    "tech": {...},      # GitHub, StackOverflow
    "news": {...},      # Reuters, AP, BBC
    "data": {...},      # World Bank, IMF
}
```

### 2. 信源可信度评分器 (SourceCredibilityScorer)

**位置**: `main.py` 第 133-220 行

**功能**:
- 四级评分：A(权威) / B(可靠) / C(一般) / D(谨慎)
- 综合评分维度：
  - 基础权重 (40%): 预定义信源权重
  - 内容质量 (30%): 数据支撑、引用、长度、作者
  - 时效性 (15%): 发布时间远近
  - 引用完整性 (15%): 引用数量和格式

**预定义信源权重**:
| 信源 | 权重 |
|------|------|
| nature.com, science.org | 0.95 |
| pubmed.ncbi.nlm.nih.gov | 0.95 |
| reuters.com, apnews.com | 0.90 |
| worldbank.org, imf.org | 0.90 |
| arxiv.org | 0.85 |
| bbc.com | 0.85 |
| github.com | 0.80 |
| stackoverflow.com | 0.75 |
| wikipedia.org | 0.70 |
| medium.com | 0.50 |
| zhihu.com | 0.45 |

### 3. 矛盾检测器 (ContradictionDetector)

**位置**: `main.py` 第 223-310 行

**功能**:
- 数值矛盾检测：相同单位但差异 > 20%
- 定性矛盾检测：对立关键词（增长/下降、支持/反对等）
- 按主题分组主张，针对性检测

### 4. 质量检查流水线 (QualityCheckPipeline)

**位置**: `main.py` 第 313-410 行

**6 维度检查**:
1. 来源数量：至少 10 个 URL
2. 引用完整性：URL + 日期
3. 数据支撑：具体数据和数值
4. 矛盾处理：矛盾/分歧章节
5. 结构完整：Summary/Finding/Source 章节
6. 语言质量：句子长度、连接词

**质量等级**:
| 分数 | 等级 | 建议 |
|------|------|------|
| ≥ 0.85 | 优秀 | 可直接交付 |
| 0.70-0.84 | 良好 | 建议人工复核 |
| 0.50-0.69 | 一般 | 建议重新研究 |
| < 0.50 | 较差 | 需重新执行 |

### 5. 领域识别与路由 (detect_domain)

**位置**: `main.py` 第 413-430 行

**支持领域**:
| 领域 | 关键词示例 |
|------|-----------|
| tech | AI, 机器学习, Python, 算法, 区块链 |
| finance | 经济, 金融, GDP, 通胀, 投资 |
| health | 医疗, 健康, 疾病, 药物, 疫苗 |
| general | 其他所有查询 |

### 6. 领域特定 Prompt

**位置**: `main.py` 第 641-730 行

`build_system_prompt()` 函数现在接受 `domain` 参数，为不同领域添加特定指令：

- **tech**: 代码示例、官方文档、版本兼容性
- **finance**: 数据日期、名义/实际值、历史背景
- **health**: 同行评审、相关性与因果、样本量

## API 变更

### ResearchAgent 初始化

```python
# 启用质量增强（默认）
agent = ResearchAgent(enable_quality_enhancement=True)

# 禁用质量增强（原始模式）
agent = ResearchAgent(enable_quality_enhancement=False)
```

### run() 方法返回增强信息

```python
result = await agent.run(query="...")

# 新增返回字段
result["domain"]          # 识别的领域: tech/finance/health/general
result["quality_report"]  # 质量检查报告
result["metadata"]        # 研究元数据（来源列表、主张列表）
```

### TaskResponse 模型扩展

```python
class TaskResponse(BaseModel):
    task_id: str
    status: str
    result: str
    # 新增字段
    domain: Optional[str]       # 领域
    quality_report: Optional[dict]  # 质量报告
    metadata: Optional[dict]    # 研究元数据
```

## 质量追踪

在研究过程中，系统自动：
1. 对每个搜索结果的 URL 进行可信度评分
2. 记录所有来源的标题、URL、评分和等级
3. 收集所有主张用于矛盾检测
4. 在最终输出前运行完整质量检查

## 使用示例

### 发起研究请求

```bash
curl -X POST http://localhost:8000/api/research \
  -H "Content-Type: application/json" \
  -d '{
    "query": "AI 机器学习最新进展",
    "max_turns": 100,
    "context_keep": 5,
    "model": "qwen-plus"
  }'
```

### 获取结果（含质量报告）

```bash
curl http://localhost:8000/api/research/{task_id}
```

响应示例：
