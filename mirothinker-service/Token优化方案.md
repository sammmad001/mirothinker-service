# MiroThinker Token 极致优化方案

> 版本: v1.0 | 2026-05-11
> 目标: 在不影响质量的前提下，减少 40-60% 的 Token 消耗

---

## 一、Token 消耗分析

### 当前单次研究的 Token 构成

```
典型研究任务 (100 轮对话):

┌─────────────────────────────────────────┐
│          Token 消耗分布                   │
├─────────────────────────────────────────┤
│                                         │
│  System Prompt:    ~800 tokens/轮       │
│  ├─ 基础指令:       400 tokens           │
│  ├─ 领域指令:       200 tokens           │
│  └─ 输出格式:       200 tokens           │
│                                         │
│  上下文历史:       ~2000 tokens/轮      │
│  ├─ 用户查询:        50 tokens           │
│  ├─ 对话历史 (5轮): 1500 tokens          │
│  └─ 工具结果:       450 tokens           │
│                                         │
│  工具调用结果:      ~500 tokens/轮       │
│  ├─ 搜索结果 (10条): 400 tokens          │
│  └─ 网页内容:        100 tokens          │
│                                         │
│  助手回复:          ~300 tokens/轮       │
│                                         │
│  总计/轮:          ~3600 tokens          │
│  100轮总计:        ~360,000 tokens       │
│                                         │
└─────────────────────────────────────────┘
```

| 优化项 | 当前消耗 | 优化后 | 减少比例 | 质量影响 |
|--------|---------|--------|---------|---------|
| System Prompt | 800 tokens | 250 tokens | **-69%** | 无 |
| 上下文历史 | 2000 tokens | 1200 tokens | **-40%** | 极小 |
| 工具结果 | 500 tokens | 250 tokens | **-50%** | 无 |
| 助手回复 | 300 tokens | 200 tokens | **-33%** | 无 |
| **总计/轮** | **3600** | **1900** | **-47%** | **极小** |

## 二、优化策略

### 策略 1: 压缩 System Prompt (减少 69%)

**优化前** (~800 tokens):
```
完整的研究流程说明、规则列表、输出格式模板
```

**优化后** (~250 tokens):
```
精简为关键指令，移除冗余说明
```

### 策略 2: 优化工具输出格式 (减少 50%)

**优化前**:
```json
[
  {"title": "完整标题...", "link": "https://...", "snippet": "很长的摘要..."},
  ...10条
]
```

**优化后**:
```
1. [标题] URL
   摘要 (50字内)
2. [标题] URL
   摘要
```

### 策略 3: 智能上下文管理 (减少 40%)

**优化前**: 固定保留最近 5 轮完整对话
**优化后**: 
- 保留关键决策点
- 压缩工具结果为摘要
- 移除重复内容

### 策略 4: 搜索结果精选 (减少 60%)

**优化前**: 返回 10 条搜索结果完整信息
**优化后**: 
- 只返回 Top 5 最相关
- 截断过长摘要
- 去除重复来源

## 五、质量保证

### 不受影响的质量维度

- ✅ 信息覆盖度 (仍搜索相同数量的源)
- ✅ 信源可信度评分 (完整评分逻辑)
- ✅ 矛盾检测 (完整检测)
- ✅ 领域适配 (仍然识别领域)
- ✅ 输出质量 (最终报告相同)

### 仅影响

- ⚠️ 上下文深度 (保留关键信息，去除冗余)
- ⚠️ 工具输出详细度 (摘要化，但不丢关键信息)

---

*优化原则: 只压缩格式，不减少信息量*
