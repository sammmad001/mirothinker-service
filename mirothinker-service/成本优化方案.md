## 2. 架构层优化

### 2.1 查询缓存机制

**问题**: 相同或相似查询重复消耗 LLM Credit

**方案**: 实现多级缓存

```
┌──────────────────────────────────────────────┐
│                  查询流程                       │
│                                              │
│  用户查询 → 语义相似度缓存 → 精确缓存 → 执行    │
│              (70%匹配)      (100%匹配)         │
└──────────────────────────────────────────────┘
```

**实现**:

```python
import hashlib
from difflib import SequenceMatcher

class QueryCache:
    def __init__(self, similarity_threshold=0.85):
        self.cache = {}           # 精确缓存
        self.semantic_cache = {}   # 语义缓存 (向量化)
        self.threshold = similarity_threshold

    def get_exact(self, query: str) -> dict | None:
        key = hashlib.md5(query.encode()).hexdigest()
        return self.cache.get(key)

    def get_similar(self, query: str, threshold=0.85) -> dict | None:
        """基于编辑距离的相似度匹配 (轻量)"""
        for cached_query, result in self.cache.items():
            if SequenceMatcher(None, query.lower(), cached_query.lower()).ratio() >= threshold:
                return result
        return None

    def set(self, query: str, result: dict):
        key = hashlib.md5(query.encode()).hexdigest()
        self.cache[key] = {
            "query": query,
            "result": result,
            "timestamp": time.time(),
        }
```

**收益**:
- 热点查询命中率预计 15-30%
- 每次命中节省: 约 2,000-5,000 Credit + $0.20 MCP 工具费
- **月节省: 约 15,000-45,000 Credit + ¥30-60**

### 3.4 MCP 工具调用优化

**问题**: Agent 可能重复调用相同工具

**方案**: 工具调用去重 + 限流

```python
class ToolCallLimiter:
    def __init__(self, max_search_per_task=15, max_scrape_per_task=5):
        self.search_count = 0
        self.scrape_count = 0
        self.searched_urls = set()
        self.max_search = max_search_per_task
        self.max_scrape = max_scrape_per_task

    def can_search(self, query: str) -> bool:
        if self.search_count >= self.max_search:
            return False
        # 去重: 相似查询不再搜索
        for past in self.searched_urls:
            if SequenceMatcher(None, query, past).ratio() > 0.9:
                return False
        return True

    def can_scrape(self, url: str) -> bool:
        if self.scrape_count >= self.max_scrape:
            return False
        if url in self.searched_urls:
            return False
        return True
```

**收益**: 减少 20-30% 冗余工具调用

## 4. 分析结构层优化

### 4.1 三阶段研究 (浅层 → 聚焦 → 交叉验证)

**问题**: 所有查询都走完整 200 轮流程，缺乏结构化控制

```
├── 4-5 次搜索 × 5-6 结果/次 = 20-25 个来源

├── LLM 初步综合 (5-6 次调用)
├── 信息充足度评估 (qwen-flash, ~20 Credit)
│
├── 充足 (25%) → 直接输出结果 (总 Credit: ~350)
│
└── 不足 (75%) → 进入 Phase 2
    │
    Phase 2: 聚焦深入 (10-15 个信息源)
    ├── 2-3 次针对性搜索
    ├── 4-6 次深度抓取
    ├── LLM 深度分析 (4-5 次调用)
    ├── 再评估
    │
    ├── 充足 → 输出结果 (总 Credit: ~750)
```

### 4.1b 三阶段 + 模型路由 (最优组合)

三阶段结构质量高但成本略高，与 Tier 路由结合可兼得：

|------|------------|----------------|------|------|

**结论**: 三阶段结构提供质量保障，路由提供成本控制，两者结合实现 **质量最优 + 成本最低**。

```python
class ThreePhaseAgent:
    def __init__(self):
        self.router = QueryRouter()
        self.cache = QueryCache()

    async def research(self, query: str) -> dict:
        # 1. 检查缓存
        cached = self.cache.get_similar(query)
        if cached:
            return cached

        # 2. Tier 路由分类
        tier = await self.router.classify(query)

        # 3. 根据层级决定策略
        if tier == "TIER_1":
            result = await self.simple_answer(query)
        elif tier == "TIER_2":
            result = await self._phase_1(query)
            if not result["sufficient"]:
                result = await self._phase_2(query, result)
        else:  # TIER_3
            result = await self._phase_1(query)
            if not result["sufficient"]:
                result = await self._phase_2(query, result)
            if not result["sufficient"]:
                result = await self._phase_3(query, result)

        # 4. 缓存结果
        self.cache.set(query, result)
        return result

    async def _phase_1(self, query: str) -> dict:
        """Phase 1: 浅层研究 (20-25 个信息源)"""
        # 并行搜索 4-5 次，每次获取 5-6 个结果
        search_queries = await self._generate_search_queries(query, count=5)
        results = await asyncio.gather(*[self.search(q) for q in search_queries])

        # 抓取关键网页
        urls_to_scrape = await self._select_urls(results, count=8)
        scraped = await asyncio.gather(*[self.scrape(u) for u in urls_to_scrape])

        # 初步综合
        synthesis = await self.llm_call(
            self._build_phase_prompt(1, query, results, scraped),
            model="qwen-plus"
        )

        # 评估是否充足
        sufficient = await self._evaluate_sufficiency(synthesis)

        return {
            "sufficient": sufficient,
            "result": synthesis,
            "sources": self._collect_sources(results, scraped),
        }

    async def _phase_2(self, query: str, prev: dict) -> dict:
        """Phase 2: 聚焦深入 (10-15 个信息源)"""
        # 基于 Phase 1 的 gap 生成针对性搜索
        gaps = await self._identify_gaps(prev["result"])
        focused_results = await self._targeted_search(query, gaps, count=15)

        # 深度分析
        deep_synthesis = await self.llm_call(
            self._build_phase_prompt(2, query, prev, focused_results),
            model="qwen-plus"
        )

        sufficient = await self._evaluate_sufficiency(deep_synthesis)

        return {
            "sufficient": sufficient,
            "result": deep_synthesis,
            "sources": prev["sources"] + self._collect_sources(focused_results),
        }

    async def _phase_3(self, query: str, prev: dict) -> dict:
        """Phase 3: 交叉验证 (2-3 个信息源)"""
        # 提取关键主张进行验证
        claims = await self._extract_claims(prev["result"])
        validations = await self._validate_claims(claims, count=3)

        # 最终综合
        final = await self.llm_call(
            self._build_phase_prompt(3, query, prev, validations),
            model="qwen-plus"
        )

        return {
            "sufficient": True,
            "result": final,
            "sources": prev["sources"],
            "validations": validations,
        }
```

### 4.2 动态早停机制 (阶段内优化)

**问题**: 每个阶段内部也可能出现信息饱和，继续搜索浪费 Credit

**方案**: 阶段内信息增益评估 + 动态提前进入下一阶段

```python
def should_early_stop(self, recent_results: list, threshold=0.1) -> bool:
    """评估最新搜索结果的信息增益"""
    if len(recent_results) < 3:
        return False

    # 比较最新结果与已有内容的重叠度
    new_content = recent_results[-1].get("snippet", "")
    existing_content = " ".join([r.get("snippet", "") for r in recent_results[:-1]])

    overlap = self._calculate_overlap(new_content, existing_content)

    # 如果重叠度 > 阈值，说明信息趋于饱和
    return overlap > threshold

def _calculate_overlap(self, new: str, existing: str) -> float:
    """计算文本重叠率"""
    new_words = set(new.lower().split())
    existing_words = set(existing.lower().split())
    if not new_words:
        return 0
    return len(new_words & existing_words) / len(new_words)
```

**收益**: 每个阶段内平均减少 20-30% 轮数，**Credit 节省 15-20%**

**与三阶段协同**: 早停机制在三阶段中更加有效——如果 Phase 1 提前饱和，直接跳到评估，跳过无谓搜索。

### 4.3 智能上下文裁剪

**当前**: 保留最近 5 条工具结果

**优化**: 基于信息量的动态裁剪

```python
def smart_context_trim(self, messages: list, target_tokens=120000) -> list:
    """智能裁剪: 保留高信息量内容"""
    # 1. 始终保留系统提示 + 用户查询
    essential = [messages[0], messages[1]]

    # 2. 按信息量排序工具结果
    tool_msgs = [m for m in messages if m.get("role") == "tool"]
    scored = []
    for msg in tool_msgs:
        content = msg.get("content", "")
        score = self._information_score(content)
        scored.append((score, msg))

    scored.sort(key=lambda x: x[0], reverse=True)

    # 3. 从高到低选取，直到达到 token 限制
    selected = []
    for score, msg in scored:
        if self._estimate_tokens(selected + [msg]) <= target_tokens:
            selected.append(msg)

    return essential + selected

def _information_score(self, text: str) -> float:
    """启发式信息量评分"""
    words = text.split()
    unique_ratio = len(set(words)) / max(len(words), 1)  # 词汇丰富度
    has_data = len([w for w in words if w.replace(".", "").isdigit()])  # 数据密度
    has_url = text.count("http")  # 引用数量
    return unique_ratio * 0.4 + min(has_data / 10, 1) * 0.3 + min(has_url / 3, 1) * 0.3
```

**收益**: 有效上下文质量提升 40%，减少不必要的 token 消耗

```python
def extract_parallel_tools(content: str) -> list[tuple]:
    """从单次 LLM 输出中提取多个工具调用"""
    import re
    tools = []

    # 提取所有 search 调用
    for match in re.finditer(r'google_search\(["\'](.+?)["\']\)', content):
        tools.append(("google_search", match.group(1)))

    # 提取所有 scrape 调用
    for match in re.finditer(r'scrape_webpage\(["\'](.+?)["\']\)', content):
        tools.append(("scrape_webpage", match.group(1)))

    return tools
```

**收益**: 减少 30-40% LLM 调用次数，**Credit 节省 30-40%**

### 5.4 响应缓存 (LLM 级别)

**问题**: 相似的中间步骤重复调用 LLM

**方案**: 基于 Prompt 哈希缓存响应

```python
class LLMCache:
    def __init__(self):
        self.cache = {}

    def get(self, messages: list) -> dict | None:
        key = hashlib.md5(json.dumps(messages, sort_keys=True).encode()).hexdigest()
        return self.cache.get(key)

    def set(self, messages: list, response: dict):
        key = hashlib.md5(json.dumps(messages, sort_keys=True).encode()).hexdigest()
        self.cache[key] = response
```

**收益**: 重复模式命中率达 10-15%，**Credit 节省 10-15%**

```python
class ThreePhaseAgent:
    def __init__(self):
        self.router = QueryRouter()
        self.cache = QueryCache()
        self.tool_limiter = ToolCallLimiter()
        self.llm_cache = LLMCache()

    async def research(self, query: str) -> dict:
        # 1. 检查缓存
        cached = self.cache.get_similar(query)
        if cached:
            return cached

        # 2. 路由分类
        tier = await self.router.classify(query)

        # 3. 根据层级决定策略
        if tier == "TIER_1":
            result = await self.simple_answer(query)
        elif tier == "TIER_2":
            result = await self._phase_1(query)
            if not result["sufficient"]:
                result = await self._phase_2(query, result)
        else:  # TIER_3
            result = await self._phase_1(query)
            if not result["sufficient"]:
                result = await self._phase_2(query, result)
            if not result["sufficient"]:
                result = await self._phase_3(query, result)

        # 4. 缓存结果
        self.cache.set(query, result)
        return result
```
