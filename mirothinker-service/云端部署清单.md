# MiroThinker 云端部署清单

## 服务器要求
- **最低配置**: 2核CPU, 2GB内存, 20GB磁盘
- **推荐配置**: 2核CPU, 4GB内存, 40GB磁盘
- **操作系统**: Ubuntu 20.04/22.04 LTS 或 CentOS 8+
- **网络**: 公网IP, 开放 80/443 端口

---

## 部署步骤

### 1. 服务器初始化
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装 Docker
curl -fsSL https://get.docker.com | sh
sudo systemctl start docker
sudo systemctl enable docker

# 安装 Docker Compose
sudo apt install docker-compose -y

# 创建部署用户 (可选，推荐)
sudo useradd -m -s /bin/bash deploy
sudo usermod -aG docker deploy
sudo usermod -aG sudo deploy
```

### 2. 上传项目文件
```bash
# 方法 1: Git Clone (推荐)
git clone <your-repo-url> ~/mirothinker-service
cd ~/mirothinker-service

# 方法 2: SCP 上传
scp -r /Users/sam/Desktop/mirothinker-service/* user@server-ip:~/mirothinker-service/

# 方法 3: 打包上传
tar czf mirothinker-service.tar.gz -C /Users/sam/Desktop mirothinker-service
scp mirothinker-service.tar.gz user@server-ip:~
ssh user@server-ip "tar xzf mirothinker-service.tar.gz && rm mirothinker-service.tar.gz"
```

### 3. 配置环境变量
```bash
cd ~/mirothinker-service

# 复制并编辑 .env
cp .env.example .env
nano .env

# 必须配置:
# DASHSCOPE_API_KEY=sk-xxx  (百炼API密钥)
```

### 4. 执行部署
```bash
cd ~/mirothinker-service

# 方法 1: 一键部署脚本 (推荐)
chmod +x cloud-deploy.sh
./cloud-deploy.sh

# 方法 2: 手动部署
docker-compose build
docker-compose up -d

# 方法 3: 2核2G服务器专用
chmod +x deploy-2core-2g.sh
./deploy-2core-2g.sh
```

### 5. 验证部署
```bash
# 检查服务状态
docker-compose ps

# 查看日志
docker-compose logs -f app
docker-compose logs -f nginx

# 测试健康检查
curl http://localhost/api/health

# 测试前端访问
curl http://localhost/

# 测试 API
curl -X POST http://localhost/api/research \
  -H "Content-Type: application/json" \
  -d '{"query": "test query", "max_turns": 5}'
```

### 6. 配置域名和 SSL (可选)
```bash
# 设置环境变量
export MIRO_DOMAIN=your-domain.com
export LETSENCRYPT_EMAIL=your@email.com

# 重新运行部署脚本 (会自动配置 SSL)
./cloud-deploy.sh

# 或手动配置
sudo apt install certbot python3-certbot-nginx
sudo certbot certonly --standalone -d your-domain.com --email your@email.com
```

## 常用管理命令

### 服务管理
```bash
cd ~/mirothinker-service

# 查看服务状态
docker-compose ps

# 停止服务
docker-compose down

# 启动服务
docker-compose up -d

# 重启服务
docker-compose restart

# 重建镜像
docker-compose build --no-cache

# 查看日志
docker-compose logs -f
docker-compose logs -f app
docker-compose logs -f nginx

# 查看实时日志 (最近100行)
docker-compose logs --tail=100 app
```

### 数据管理
```bash
# 执行备份
chmod +x backup.sh
./backup.sh

# 查看备份列表
ls -lh ~/backups/mirothinker/

# 恢复备份
tar xzf ~/backups/mirothinker/mirothinker_backup_YYYYMMDD_HHMMSS.tar.gz -C /tmp/
cp -r /tmp/backup_YYYYMMDD_HHMMSS/* ~/mirothinker-service/
docker-compose restart
```

### 更新代码
```bash
# Git 更新
cd ~/mirothinker-service
git pull
docker-compose build
docker-compose up -d

# 或运行部署脚本
./cloud-deploy.sh
```

### 资源监控
```bash
# 查看容器资源使用
docker stats mirothinker-service mirothinker-nginx

# 查看磁盘使用
du -sh ~/mirothinker-service/{traces,logs,cache}

# 清理 Docker 无用资源
docker system prune -af
```

## 故障排查

### 服务无法启动
```bash
# 查看详细日志
docker-compose logs app

# 检查端口占用
sudo netstat -tlnp | grep -E '80|443|8000'

# 检查 Docker 状态
sudo systemctl status docker
```

### API 无法访问
```bash
# 检查后端健康状态
curl http://localhost:8000/api/health

# 直接访问后端 (绕过 Nginx)
docker exec -it mirothinker-service curl http://localhost:8000/api/health

# 检查 Nginx 配置
docker exec -it mirothinker-nginx nginx -t
docker exec -it mirothinker-nginx cat /etc/nginx/nginx.conf
```

### 前端无法访问
```bash
# 检查 Nginx 日志
docker exec -it mirothinker-nginx cat /var/log/nginx/error.log

# 检查静态文件
docker exec -it mirothinker-nginx ls -la /usr/share/nginx/html/

# 重启 Nginx
docker-compose restart nginx
```

### 内存不足
```bash
# 查看内存使用
free -h

# 调整 Docker 资源限制
nano docker-compose.yml
# 修改 memory 限制

# 重启服务
docker-compose down
docker-compose up -d
```

## 安全建议

1. **防火墙配置**
```bash
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 22/tcp
sudo ufw enable
```

2. **API 密钥保护**
- 永远不要将 `.env` 文件提交到 Git
- 使用强 API Key
- 定期轮换 API Key

3. **SSL/TLS**
- 生产环境必须启用 HTTPS
- 使用 Let's Encrypt 免费证书
- 配置自动续期: `sudo certbot renew --dry-run`

4. **访问限制**
- 配置 Nginx 速率限制 (已在 nginx.conf 中配置)
- 考虑添加 API 认证
- 定期查看访问日志

## 性能优化

### 当前配置
- **Workers**: 2 (docker-compose.yml)
- **并发限制**: 10 (docker-compose.yml)
- **CPU 限制**: 1.5 cores (docker-compose.yml)
- **内存限制**: 3GB (docker-compose.yml)

### 调优建议
1. **2核2G 服务器**: 使用 `deploy-2core-2g.sh`
2. **4核4G 服务器**: 可增加 workers 到 4
3. **高并发场景**: 添加 Redis 缓存层
4. **大量研究任务**: 使用消息队列 (RabbitMQ/Celery)

## 备份策略

### 自动备份 (推荐配置 cron)
```bash
# 编辑 crontab
crontab -e

# 每天凌晨 2 点备份
0 2 * * * cd ~/mirothinker-service && ./backup.sh

# 每周清理旧备份 (保留 30 天)
0 3 * * 0 find ~/backups/mirothinker -name "*.tar.gz" -mtime +30 -delete
```

### 备份内容
- traces/ (研究追踪数据)
- logs/ (应用日志)
- cache/ (缓存数据)
- benchmarks/ (基准测试数据)
- 配置文件 (docker-compose.yml, nginx.conf, .env)
