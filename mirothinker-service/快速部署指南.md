# MiroThinker 快速部署指南 — 5分钟上云

> 版本: v2.0 | 2026-05-11

## 前提条件

- 阿里云 ECS 实例 (2核4G 或以上)
- 域名 (可选，用于 HTTPS)
- 以下 API 密钥:
  - 阿里云百炼 DashScope API Key
  - Serper API Key
  - Jina API Key

---

## 方式一: Docker Compose 部署 (推荐)

### 1. 上传代码

```bash
# 在本地执行
scp -r mirothinker-service/ root@YOUR_SERVER_IP:/opt/
```

### 2. SSH 到服务器

```bash
ssh root@YOUR_SERVER_IP
cd /opt/mirothinker-service
```

### 3. 安装 Docker (如未安装)

```bash
curl -fsSL https://get.docker.com | sh
sudo systemctl enable docker
sudo systemctl start docker
```

### 4. 配置环境变量

```bash
cp .env.example .env
nano .env
```

填入你的 API 密钥:
```env
DASHSCOPE_API_KEY=sk-xxxxx
SERPER_API_KEY=xxxxx
JINA_API_KEY=jina_xxxxx
```

### 5. 一键启动

```bash
chmod +x deploy.sh
./deploy.sh
# 选择 1) Docker Compose
```

### 6. 验证

```bash
curl http://localhost:8000/api/health
# 应返回: {"status": "healthy", "dashscope_configured": true, ...}
```

### 7. (可选) 配置 Nginx + HTTPS

```bash
# 安装 Nginx
apt install nginx -y

# 配置反向代理
cp nginx.conf /etc/nginx/conf.d/mirothinker.conf
nano /etc/nginx/conf.d/mirothinker.conf
# 修改 server_name 为你的域名

# 配置 HTTPS (Let's Encrypt)
apt install certbot python3-certbot-nginx -y
certbot --nginx -d your-domain.com

# 重启 Nginx
systemctl restart nginx
```

## 方式二: 直接 Python 部署

### 1-3. 同上 (上传代码、SSH、配置 .env)

### 4. 安装依赖并启动

```bash
cd /opt/mirothinker-service
./deploy.sh
# 选择 2) Direct Python Deployment
```

或手动执行:

```bash
cd /opt/mirothinker-service/backend
pip3 install -r requirements.txt

# 加载环境变量
set -a
source ../.env
set +a

# 启动服务
nohup python3 -m uvicorn main:app \
    --host 0.0.0.0 \
    --port 8000 \
    --workers 2 \
    > /var/log/mirothinker.log 2>&1 &
```

## 常用运维命令

### Docker 模式

```bash
# 查看日志
docker-compose logs -f

# 重启服务
docker-compose restart

# 停止服务
docker-compose down

# 更新代码后重新部署
git pull  # 或重新上传代码
docker-compose build --no-cache
docker-compose up -d
```

### 直接部署模式

```bash
# 查看日志
tail -f /var/log/mirothinker.log

# 重启服务
pkill -f "uvicorn main:app"
nohup python3 -m uvicorn main:app --host 0.0.0.0 --port 8000 --workers 2 > /var/log/mirothinker.log 2>&1 &

# 查看进程
ps aux | grep uvicorn
```

## API 使用示例

### 1. 发起研究任务

```bash
curl -X POST http://YOUR_SERVER/api/research \
  -H "Content-Type: application/json" \
  -d '{
    "query": "AI 大模型最新进展",
    "max_turns": 100,
    "model": "qwen-plus"
  }'
```

返回:
```json
{
  "task_id": "a1b2c3d4"
}
```

### 2. 查询结果

```bash
curl http://YOUR_SERVER/api/research/a1b2c3d4
```

### 3. 健康检查

```bash
curl http://YOUR_SERVER/api/health
```

## 故障排查

### 服务无法启动

```bash
# 检查端口占用
netstat -tlnp | grep 8000

# 检查日志
docker-compose logs     # Docker 模式
tail -f /var/log/mirothinker.log  # 直接模式
```

### API 调用失败

```bash
# 检查环境变量
cat .env

# 测试 API 连接
curl -v http://localhost:8000/api/health
```

### 内存不足

```bash
# 查看内存使用
free -h

# 减少 worker 数量
# 编辑 docker-compose.yml 或启动命令，将 --workers 改为 1
```
