## 部署选项对比

| 维度 | Docker Compose | 直接 Python |
|------|----------------|-------------|
| **部署速度** | ⭐⭐⭐⭐⭐ 一键启动 | ⭐⭐⭐ 需手动配置 |
| **环境一致性** | ⭐⭐⭐⭐⭐ 完全隔离 | ⭐⭐⭐ 依赖系统 Python |
| **回滚能力** | ⭐⭐⭐⭐⭐ 镜像版本管理 | ⭐⭐ 需手动备份 |
| **资源占用** | ⭐⭐⭐ 额外 Docker 开销 (~200MB) | ⭐⭐⭐⭐⭐ 最小化 |
| **调试便利** | ⭐⭐⭐ 需 docker logs | ⭐⭐⭐⭐⭐ 直接查看 |
| **推荐场景** | **生产环境** | 测试/开发环境 |

**建议**: 生产环境使用 Docker Compose 部署。

## 快速开始

```bash
# 1. 上传到服务器
scp -r mirothinker-service/ root@YOUR_SERVER:/opt/

# 2. SSH 到服务器
ssh root@YOUR_SERVER
cd /opt/mirothinker-service

# 3. 配置并部署
cp .env.example .env && nano .env  # 填入 API 密钥
chmod +x deploy.sh
./deploy.sh                        # 选择 Docker Compose

# 4. 验证
curl http://localhost:8000/api/health
```
