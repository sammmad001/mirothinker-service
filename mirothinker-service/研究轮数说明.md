# MiroThinker 研究轮数设计说明

> 为什么需要 50+40+20+10 轮？可以调整吗？

---

## 一、研究轮数详解

### 标准研究流程 (默认 120 轮)

```
完整研究流程:

Phase 1: 浅层研究 (50 轮)
├─ 目的: 广泛收集信息，了解全局
├─ 搜索: 20-25 个不同关键词
├─ 抓取: 10-15 个网页详情
└─ 产出: 信息全景图，识别关键方向

Phase 2: 聚焦研究 (40 轮)  
├─ 目的: 深入关键领域
├─ 搜索: 10-15 个精准关键词
├─ 抓取: 8-10 个深度内容
└─ 产出: 核心发现，数据支撑

Phase 3: 交叉验证 (20 轮)
├─ 目的: 验证关键主张
├─ 搜索: 5-8 个验证性搜索
├─ 抓取: 3-5 个原始来源
└─ 产出: 矛盾检测，可信度评估

Phase 4: 报告生成 (10 轮)
├─ 目的: 整合所有发现
├─ 整理: 结构化输出
└─ 产出: 最终研究报告

总计: 120 轮
```

### 每轮做什么？

```
每轮对话 = 1 次 LLM 调用

LLM 思考: "我需要搜索什么？"
  ↓
执行工具: google_search("关键词")
  ↓
获得结果: 5-10 条搜索摘要
  ↓
LLM 思考: "我需要抓取哪个网页？"
  ↓
执行工具: scrape_webpage("URL")
  ↓
获得内容: 网页详情 (1500 chars)
  ↓
LLM 思考: "我还需要什么信息？"
  ↓
重复...直到信息充分
  ↓
输出: FINAL ANSWER (研究报告)
```

## 二、为什么需要这么多轮？

### 研究深度 vs 轮数关系

| 轮数 | 研究深度 | 适用场景 | 质量 |
|------|---------|---------|------|
| 10-20 | 表面概览 | 简单事实查询 | ⭐⭐ |
| 30-50 | 基本了解 | 一般性问题 | ⭐⭐⭐ |
| 80-120 | 深度研究 | 产业分析、投资决策 | ⭐⭐⭐⭐⭐ |
| 150-200 | 极致深入 | 学术论文、尽职调查 | ⭐⭐⭐⭐⭐⭐ |

### 轮数不足的后果

```
轮数过少 (如 30 轮):
├─ 搜索源: 8-10 个 (不够全面)
├─ 网页抓取: 3-5 个 (深度不足)
├─ 交叉验证: 无 (无法发现矛盾)
└─ 报告质量: 基于有限信息，可能有偏差

轮数充足 (120 轮):
├─ 搜索源: 25-35 个 (全面覆盖)
├─ 网页抓取: 15-20 个 (深度分析)
├─ 交叉验证: 有 (矛盾检测)
└─ 报告质量: 多源验证，可信度高
```

## 三、如何调整研究轮数？

### 方法 1: API 调用时指定 (推荐)

```bash
# 快速研究 (30 轮, 适合简单问题)
curl -X POST http://127.0.0.1:8000/api/research \
  -H "Content-Type: application/json" \
  -d '{
    "query": "什么是 CPO 技术？",
    "max_turns": 30
  }'

# 标准研究 (80 轮, 适合一般分析)
curl -X POST http://127.0.0.1:8000/api/research \
  -d '{
    "query": "分析光模块产业竞争格局",
    "max_turns": 80
  }'

# 深度研究 (150 轮, 适合投资决策)
curl -X POST http://127.0.0.1:8000/api/research \
  -d '{
    "query": "光模块、CPO和存储产业深度分析",
    "max_turns": 150
  }'
```

### 方法 2: 修改默认配置

编辑 `backend/main.py`:

```python
# 找到 ResearchRequest 类
class ResearchRequest(BaseModel):
    query: str
    max_turns: int = 200    # ← 改这里 (默认 200)
    context_keep: int = 5   # 上下文保留轮数
    model: str = "qwen-plus"
    temperature: float = 0.7
```

### 方法 3: 前端界面调整

在前端表单中添加"研究深度"选项:

```html
<select id="maxTurns">
  <option value="30">快速 (30轮, ~1分钟)</option>
  <option value="80">标准 (80轮, ~3分钟)</option>
  <option value="150" selected>深度 (150轮, ~6分钟)</option>
  <option value="200">极致 (200轮, ~8分钟)</option>
</select>
```

## 五、推荐配置

### 按场景选择轮数

| 场景 | 推荐轮数 | 原因 |
|------|---------|------|
| **简单事实查询** | 20-30 | 如"什么是 HBM？" |
| **概念解释** | 30-50 | 如"CPO 技术原理" |
| **产业概览** | 50-80 | 如"存储产业概况" |
| **深度分析** | 80-120 | 如"光模块产业竞争格局" |
| **投资决策** | 120-150 | 如"投资标的深度分析" |
| **尽职调查** | 150-200 | 如"并购标的尽调" |

### 成本优化策略

```
日常使用建议:

1. 先用 30 轮快速了解
   ↓ (如果不够)
2. 再用 80 轮深入研究
   ↓ (如果需要)
3. 最后 150 轮极致验证

总成本: 456 + 1,216 + 1,824 = 3,496 Credits
比直接 200 轮 (3,040 Credits) 略高，但更灵活
```

## 七、总结

### 轮数设计原则

1. **不是固定的**: 可以根据需要调整
2. **质量与成本权衡**: 越多轮 = 越深入 = 越贵
3. **智能停止**: Agent 会在信息充分时自动停止 (输出 FINAL ANSWER)
4. **实际使用 < max_turns**: 设置 150 轮，通常 80-120 轮就会完成

### 推荐设置

```
默认配置: max_turns = 100

理由:
• 足够完成深度研究
• 成本可控 (~¥15/次)
• 免费额度支持 ~60 次/月
• 质量有保证
```

### 调整建议

- **测试阶段**: 30-50 轮 (省钱)
- **日常使用**: 80-100 轮 (平衡)  
- **重要决策**: 120-150 轮 (深度)
- **极限研究**: 150-200 轮 (极致)

---

*轮数完全可控，按需调整！*
